安全防御

自动封禁规则说明

检测范围

监控所有网站的 access.log 和 error.log

触发条件（命中以下任一规则计为 1 次违规）

  • POST /wp-login.php — WordPress 登录暴力破解
  • POST /xmlrpc.php — XML-RPC 接口攻击
  • 访问敏感文件返回 404 — 扫描 .env、.git、wp-config.php、.sql、.tar.gz、.zip 等

封禁阈值（可在安全设置中自定义）

默认：60 秒内累计 5 次违规 → 自动封禁

多级渐进封禁

  • 临时10分钟 — 初犯：Fail2ban 封禁 10 分钟，到期自动解除
  • 临时24小时 — 累犯：解封后 24 小时内再次违规 → 升级为 24 小时，同时写入 Nftables 持久规则
  • 永久封禁 — 惯犯：累计 3 次 24 小时封禁 → 自动转为永久封禁，仅通过 Nftables 持久规则封锁

封禁机制

  • Fail2ban — 管理临时封禁（10分钟/24小时），到期自动释放
  • Nftables 持久 — 管理永久封禁，独立于 Fail2ban，重启不丢失

白名单豁免

127.0.0.1、Cloudflare / Googlebot / Bingbot 官方 IP、自定义白名单 IP 不会被封禁

服务器验证命令

SSH 登录服务器执行以下命令查看当前封禁状态：

# 查看 Fail2ban 临时封禁
fail2ban-client status wppanel
# 查看 Nftables 持久封禁
nft list set ip wppanel_persist banned_ips
