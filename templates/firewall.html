安全防御

安全防御规则说明

检测范围

监控所有网站的 access.log 和 error.log 日志。

触发条件（命中以下任一规则计为 1 次违规）

  • POST /wp-login.php — WordPress 登录暴力破解
  • POST /xmlrpc.php — XML-RPC 接口攻击（含 //xmlrpc.php 变体）
  • 返回 HTTP 429 — 超出请求频率限制
  • 敏感文件扫描返回 404 — 如 .env、.git、wp-config.php、.sql、压缩包等
  • 404 泛滥检测 — 任意路径返回 404 达 30 次/60 秒 判定为目录扫描（独立规则，固定阈值）

封禁阈值

前 4 项规则默认 60 秒内累计 5 次违规 → 自动封禁（可在「安全设置」页面自定义阈值、窗口和封禁时长）。

404 泛滥规则固定为 30 次/60 秒，不开放自定义。

多级渐进封禁

  • 首次违规 → Fail2ban 封禁 10 分钟，到期自动解除
  • 24 小时内再次违规 → 升级为 24 小时 封禁，同时写入 Nftables 持久规则
  • 累计 3 次 24 小时封禁 → 自动转为永久封禁，仅通过 Nftables 持久规则封锁

封禁列表可通过「来源」列区分 Fail2ban 临时封禁 与 Nftables 持久封禁。

封禁机制

  • Fail2ban — 管理网站防护封禁（jail: wppanel、wppanel-404）和 SSH 防护封禁（jail: wppanel-sshd），到期自动释放
  • Nftables 持久 — 管理永久封禁、扫描防御封禁（来源: panel_scan）和面板认证封禁（来源: panel），独立于 Fail2ban，到期自动解除

扫描防御

  • 针对面板 8443 端口的扫描防护，不作用于网站端口（80/443）
  • 任何非浏览器特征请求（如 curl、脚本、扫描器）访问面板端口且未命中随机入口 → 封禁 30 天
  • 命中随机入口（/{suffix}）的请求正常放行，不受影响

触发后写入面板封禁列表，来源标注为「扫描防御」，等级为临时 30 天。

白名单豁免

127.0.0.1、Cloudflare / Googlebot / Bingbot 官方 IP 段、自定义白名单 IP 不会被封禁。白名单在「安全设置」页面管理。

请求频率限制原理

采用令牌桶算法：每 IP 一个桶，容量 = 突发缓冲（默认 300），每秒补充 1 个令牌。请求到达消耗令牌，耗尽后返回 429。人类浏览有自然停顿，停顿期间桶自动回满；机器人持续请求则被压制在 1 次/秒。

⚠ 多次 429 会被 Fail2ban 自动封禁。请确保突发缓冲足够日常使用。

✓ 提高突发缓冲不影响拦截机器人，不削弱安全防护。

服务器验证命令

SSH 登录服务器执行以下命令查看当前封禁状态：

# 查看网站防护封禁
fail2ban-client status wppanel
# 查看 404 泛滥封禁
fail2ban-client status wppanel-404
# 查看 SSH 暴力破解封禁
fail2ban-client status wppanel-sshd
# 查看 Nftables 持久封禁（扫描防御 + 面板认证封禁）
nft list set ip wppanel_persist banned_ips
