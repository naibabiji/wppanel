安全设置说明

Fail2ban 配置项

  • 触发阈值（次）

    在统计窗口内，单个 IP 触发特征规则的次数达到此值即自动封禁。默认 5 次。调低会更敏感（容易误封），调高则更宽松。

  • 统计窗口（秒）

    用于累计违规次数的时间范围。默认 60 秒。与触发阈值配合使用：例如默认配置即"60 秒内违规 5 次则封禁"。窗口越大，累计越容易触发。

  • 封禁时长（秒）

    首次违规触发封禁后，IP 被防火墙拦截的时长。默认 600 秒（10 分钟）。到期后 fail2ban 自动解封，但若该 IP 在 24 小时内再次违规，将自动升级为 24 小时封禁。

白名单机制

  • 官方白名单

    面板自动从以下官方来源拉取最新 IP 段，加入 fail2ban 白名单，防止误封合法爬虫和 CDN 节点：

    • Cloudflare — 全球 CDN 和反向代理 IP 段（ipv4 + ipv6）
    • Googlebot — Google 搜索爬虫官方 IP 段
    • Bingbot — Bing 搜索爬虫官方 IP 段

    勾选"每周自动更新"后，系统将在每周一凌晨 4:00 自动重新拉取最新 IP。

  • 自定义白名单

    手动添加不会被封禁的 IP 或 IP 段，每行一个。支持 IPv4/IPv6 地址和 CIDR 格式（如 10.0.0.0/8）。自定义白名单不会被官方拉取覆盖。

渐进封禁与双机制

  • 首次违规 → Fail2ban 封禁 10 分钟，到期自动解除
  • 24 小时内再次违规 → 升级为 24 小时封禁，同时写入 Nftables 持久规则
  • 累计 3 次 24 小时封禁 → 自动转为永久封禁，仅通过 Nftables 持久封锁

封禁列表页可通过"来源"列区分 Fail2ban 临时封禁 与 Nftables 持久封禁。

服务器验证命令

SSH 登录服务器执行以下命令查看当前封禁状态：

# Fail2ban 临时封禁状态
fail2ban-client status wppanel
# Nftables 持久封禁列表
nft list set ip wppanel_persist banned_ips

Fail2ban 配置

请求频率限制

开启后每个 IP 分钟请求数受限，超限返回 429，触发多次 429 将被 Fail2ban 封禁

正常浏览约 30-50 次/分钟。攻击行为通常远超此值。建议 30-120

瞬间超出限制的额外允许量，页面含多图片/CSS/JS 时短期请求可能集中
